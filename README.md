# 🚢 세계로 무역 게임

교실에서 여러 학생(또는 모둠)이 각자 기기로 접속해 함께 하는 **실시간 온라인 무역 게임**입니다.
교사가 방을 열면 학생들이 6자리 코드로 입장하고, 나라(또는 한국의 지역)를 맡아 자원을 생산·거래·가공하며 이익을 겨룹니다.

---

> ### 📌 다른 선생님들께 나눠줄 때
> **선생님마다 설정할 필요가 없습니다.** 한 사람이 아래 [준비하기](#준비하기-firebase-설정)와 [배포](#인터넷에-올리기-배포)를 **딱 한 번** 해두고 주소만 알려주면,
> 나머지 선생님들은 그 주소에 들어가 방을 만들기만 하면 됩니다. 설치도, 가입도, 설정도 없습니다.

## 게임 소개

### 두 가지 무대
| 모드 | 참가 단위 | 예시 |
|---|---|---|
| 🌍 **세계 국가** | 14개국 | 한국·미국·사우디·브라질·칠레… |
| 🇰🇷 **한국 지역** | 17개 시/군/구 또는 17개 시/도 | 제주·안동·청주·부산 / 서울·경기·부산광역시… |

한국 지역 모드는 방을 만들 때 **단위**를 한 번 더 고릅니다.
- 🏘️ **시·군·구** — 제주·안동·청주 같은 좁은 단위. 지역색이 뚜렷한 특산물 위주
- 🗺️ **시·도** — 서울·경기·부산광역시 같은 넓은 단위. 더 큰 그림에서 무역을 배울 때

### 세 가지 난이도
| 난이도 | 다루는 자원 | 특징 |
|---|---|---|
| 🌱 **쉬움** | 원자재(특산품)만 | 제작 없이 사고팔기에만 집중. 저학년·첫 수업용 |
| 🌿 **보통** | 원자재 + 1차 가공품 | 자원을 조합해 가공품을 만들 수 있음 |
| 🌳 **어려움** | 원자재 + 1차 + 완제품 | 가공품을 다시 조합해 완제품까지. 긴 생산 사슬 |

### 핵심 재미
- **특산품 자동 생산** — 나라마다 매 턴 다른 자원이 쌓입니다
- **협상 무역** — 필요한 자원은 다른 나라와 합의해서 사고팝니다
- **자원 가공** — 재료를 모아 조합하면 훨씬 비싼 물건이 됩니다 (부가가치 학습)
- **이벤트 카드 128종** — 실제 사회 현상(AI 열풍, 유가 급등, 김장철, 저출산 등)을 반영하며, 낮은 확률로 엉뚱한 재미 이벤트도 등장합니다. **한 게임에서 같은 카드는 두 번 나오지 않습니다**
- **모둠 전원 참여** — 역할 배정 + 팀 과반 투표 + 모둠 회의실 + 학생별 활동량 집계

## 🧪 먼저 혼자 해보기 (설정 없이)

**Firebase 설정을 하기 전에도 게임을 통째로 체험해 볼 수 있습니다.**
로컬 서버를 띄운 뒤 `preview/index.html`로 들어가면, 교사 화면과 학생 화면 여러 개가 한 화면에 나란히 뜹니다.
혼자서 방을 만들고, 학생으로 입장해 거래하고, 턴을 진행하는 전 과정을 연습할 수 있어요.

```bash
python -m http.server 8000
# 브라우저에서 http://localhost:8000/preview/index.html
```

- **＋ 학생 창 추가** 버튼으로 모둠 인원을 늘려 팀 투표까지 시험해 볼 수 있습니다
- 이 창 안에서만 돌아가며 **새로고침하면 데이터가 사라집니다** (실제 수업에는 쓸 수 없어요)
- 실제 수업에서 학생들이 각자 기기로 접속하려면 아래 Firebase 설정이 필요합니다

## 준비하기 (Firebase 설정)

학생들이 서로 연결되려면 데이터를 주고받을 곳이 필요합니다. 구글의 **Firebase**를 쓰면 무료이고, 교실 규모에서는 요금이 나오지 않습니다. **10분이면 끝납니다.**

> 이 과정은 **한 사람이 딱 한 번만** 하면 됩니다. 배포한 주소를 받아 쓰는 다른 선생님들은 이 장을 건너뛰세요.

### 1단계 · 프로젝트 만들기
1. [Firebase 콘솔](https://console.firebase.google.com)에 구글 계정으로 로그인합니다.
2. **프로젝트 만들기** 클릭 → 이름은 아무거나 (예: `trade-game`).
3. Google 애널리틱스는 **사용 안 함**으로 해도 됩니다. → **프로젝트 만들기**.

### 2단계 · 데이터베이스 만들기
1. 왼쪽 메뉴에서 **빌드 → Realtime Database** 를 클릭합니다.
   > ⚠️ **Firestore Database**가 아니라 **Realtime Database**입니다. 헷갈리기 쉬우니 확인하세요!
2. **데이터베이스 만들기** 클릭.
3. 위치는 기본값(또는 `asia-southeast1`)을 고릅니다.
4. 보안 규칙은 **테스트 모드로 시작**을 선택합니다. → **사용 설정**.

### 3단계 · 설정값 복사하기
1. 왼쪽 위 톱니바퀴 ⚙️ → **프로젝트 설정**.
2. 아래로 내려 **내 앱** 영역에서 웹 아이콘 **`</>`** 을 클릭합니다.
3. 앱 닉네임을 적고 **앱 등록** (호스팅 설정은 체크하지 않아도 됩니다).
4. 화면에 이런 코드가 나옵니다:
   ```js
   const firebaseConfig = {
     apiKey: "AIzaSy...",
     authDomain: "trade-game-xxxx.firebaseapp.com",
     databaseURL: "https://trade-game-xxxx-default-rtdb.firebaseio.com",
     ...
   };
   ```
5. 이 값을 그대로 복사해 이 폴더의 **`js/firebase-config.js`** 파일 안 `firebaseConfig` 부분에 **덮어씁니다**.

> **`databaseURL`이 없다면?** 2단계(Realtime Database 만들기)를 건너뛴 것입니다. 다시 만들고 설정값을 확인하세요.

### 4단계 · 테스트 모드 기간 늘리기 (권장)
테스트 모드는 30일 후 잠깁니다. 계속 쓰려면:
**Realtime Database → 규칙** 탭에서 아래처럼 바꾸고 **게시**하세요.

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

> 🔒 **보안 안내**: 이 규칙은 주소를 아는 누구나 데이터를 읽고 쓸 수 있습니다. 수업용 게임이고 개인정보(학생 이름 외)를 저장하지 않으므로 실용적인 선택이지만, 민감한 정보는 절대 넣지 마세요. 수업이 끝나면 교사 화면의 **방 삭제**로 데이터를 지울 수 있습니다.

## 인터넷에 올리기 (배포)

### 방법 A · 로컬에서만 (같은 교실 Wi-Fi)
파일을 브라우저에서 바로 열면 안 됩니다(모듈 보안 정책). 간단한 서버가 필요합니다.

```bash
# Python이 있다면
python -m http.server 8000
```
그 다음 `http://localhost:8000` 접속. 학생들은 교사 컴퓨터의 IP(예: `http://192.168.0.5:8000`)로 접속합니다.

### 방법 B · Vercel (추천, 무료·어디서나 접속 가능)
1. [vercel.com](https://vercel.com)에 깃허브/구글 계정으로 가입합니다.
2. 이 폴더를 통째로 드래그해서 올리거나, GitHub 저장소를 연결합니다.
3. 빌드 설정은 필요 없습니다 (그냥 정적 파일). **Deploy** 클릭.
4. 받은 주소(예: `https://trade-game.vercel.app`)를 **다른 선생님들과 학생들에게 알려주면 끝입니다.**

> ⚠️ 배포하기 전에 `js/firebase-config.js`에 설정값이 들어 있는지 꼭 확인하세요.
> 빠져 있으면 접속한 사람 모두에게 설정 안내 화면이 뜹니다.

### 방법 C · Firebase Hosting
```bash
npm install -g firebase-tools
firebase login
firebase init hosting   # public 디렉터리는 . 으로, SPA는 아니오
firebase deploy
```

## 여러 선생님이 함께 쓰기

한 번 배포해 둔 주소는 **여러 선생님이 동시에 써도 됩니다.** 방 코드가 서로 다르면 각자 다른 게임이 돌아갑니다.
학교나 지역에서 함께 쓰실 때 알아두면 좋은 점들입니다.

### 동시 접속은 100명까지
Firebase 무료 요금제(Spark)는 [동시 접속 100명](https://firebase.google.com/docs/database/usage/limits)이 한도입니다.
한 학급 30명이면 **세 반이 같은 시간에 수업해도 넉넉합니다.** 다만 학교 전체가 같은 교시에 몰리면 접속이 거부될 수 있습니다.

더 많이 필요하면 Firebase 콘솔에서 **Blaze(종량제)** 로 올리면 동시 접속 20만 명까지 늘어납니다.
카드 등록이 필요하지만, 이 게임 정도의 사용량으로는 **실제 요금이 거의 0원**입니다. (무료 한도가 그대로 유지되고 초과분만 과금됩니다)

### 방은 3일 뒤 자동으로 지워집니다
여러 사람이 한 주소를 쓰면 지난 수업의 방이 쌓입니다.
그래서 누군가 교사 화면에 들어올 때마다 **만든 지 3일이 지난 방을 자동으로 정리**합니다.
수업 기록을 남기고 싶다면 게임이 끝난 뒤 결과 화면을 캡처해 두세요.

### 학생이 값을 조작하면 교사 화면에 경고가 뜹니다
데이터베이스가 열려 있어서, 브라우저 개발자도구를 다룰 줄 아는 학생은 이론상 자기 나라 돈을 바꿀 수 있습니다.
그래서 나라의 돈과 자원은 **턴 진행과 교사 조정으로만 바뀐다**는 점을 이용해 감시합니다.

협상 중에 값이 달라지면 교사 화면 맨 위에 빨간 경고가 뜨고, 어느 나라가 무엇을 얼마나 바꿨는지 보여줍니다.
**「원래대로 되돌리기」** 버튼을 누르면 마지막 턴 종료 시점의 값으로 복구되고 기록에도 남습니다.

> 완벽한 차단은 아닙니다. 아주 집요한 학생은 우회할 수 있어요. 다만 교실에서 몰래 넘어가는 일은 사실상 막아줍니다.

## 수업에서 진행하는 법

### 교사
1. 주소 접속 → **방 만들기**
2. **설정 진행**: 무대(세계/한국) → (한국이면 시·군·구/시·도 단위) → 난이도 → 참가할 나라·지역 목록 다듬기 → 규칙(시작 자금, 목표 턴, 팀 투표 여부)
   - 💡 **모둠 수만큼만 나라를 남기세요.** 나라 수가 모둠 수보다 많으면 빈 나라가 생깁니다.
3. 화면에 뜬 **6자리 코드**를 칠판에 적습니다.
4. 학생들이 다 들어오면 **게임 시작!**
5. 학생들이 협상하는 동안 기다렸다가, 충분히 진행되면 **⏭️ 턴 진행하기**를 누릅니다.
6. 중간중간 **🎲 이벤트 카드**를 발동해 시세를 흔들어 주세요. (랜덤 뽑기 또는 직접 선택)
7. 목표 턴에 도달하면 **🏁 게임 종료** → 최종 순위 발표.

### 학생
1. 주소 접속 → 코드와 이름 입력
2. 우리 나라 선택 (친구가 이미 들어간 나라를 고르면 **같은 모둠**, 인원 제한 없음)
3. 역할 선택 (무역대표/생산부장/재무관/정보분석가 — 참고용이며 모든 활동은 누구나 가능)
4. 다른 나라와 거래를 제안하고, 받은 제안에 팀원과 함께 투표해 결정합니다
5. 제작소에서 자원을 조합해 부가가치를 만듭니다

### 수업 시간 배분 예시 (45분)
| 시간 | 활동 |
|---|---|
| 5분 | 규칙 설명, 방 코드 안내, 입장 |
| 5분 | 모둠별 전략 회의 (회의실 탭 활용) |
| 25분 | 5~7턴 진행 (턴당 3~4분) |
| 5분 | 결과 발표 |
| 5분 | 마무리 토의 — "왜 그 나라가 이겼을까?", "가공을 하니 무엇이 달라졌나?" |

## 게임 규칙 자세히

### 한 턴의 흐름
```
[협상 단계]  학생들이 자유롭게 거래 제안 · 수락 · 제작 예약
      ↓  교사가 「턴 진행」 클릭
① 합의된 거래 체결   (자원 ↔ 돈 교환. 재고·자금이 부족하면 불발되고 기록에 남음)
② 특산품 자동 생산   (모든 나라가 설정된 양만큼 자원 획득)
③ 예약된 제작 실행   (이번 턴에 사 온 재료도 사용 가능!)
      ↓
[다음 턴 협상 시작]
```

### 거래가 성사되기까지 (팀 투표 켠 경우)
```
누군가 제안 작성
   → 우리 모둠 과반 찬성  → 상대에게 전달
   → 상대 모둠 과반 찬성  → ✅ 합의 완료
   → 턴 진행 시 체결
```
반대가 과반이 되면 즉시 거절됩니다. 팀 투표를 끄면 누구든 혼자 결정할 수 있습니다.

### 점수 (총자산)
```
총자산 = 보유 현금 + Σ(보유 자원 수량 × 현재 시세)
```
자원을 쌓아만 둬도 자산이지만, **가공하면 훨씬 커집니다.** 시세는 이벤트로 변하므로 비싼 자원을 언제 파느냐도 중요합니다.

### 밸런스
모든 나라의 **턴당 생산 가치**를 약 5만원 수준으로 맞춰 두었습니다. 특정 나라가 처음부터 유리하지 않으며, 승부는 **협상력과 가공 전략**에서 갈립니다.

## 내용 수정하기

| 무엇을 | 어디를 |
|---|---|
| 자원 종류·가격·단위 | `js/presets.js` 의 `WORLD_RESOURCES`, `KOREA_RESOURCES` |
| 제작 레시피 | `js/presets.js` 의 `WORLD_RECIPES`, `KOREA_RECIPES` |
| 나라·지역과 특산품 | `js/presets.js` 의 `WORLD_NATIONS`, `KOREA_NATIONS` |
| 이벤트 카드 | `js/events.js` |
| 시작 자금·기본 턴 수 | `js/presets.js` 의 `DEFAULT_SETTINGS` |
| 화면 색상·글꼴 | `css/style.css` 맨 위 `:root` (색·모서리·그림자를 한곳에서 관리) |
| 자원 카드·순위 시상대 등 공용 부품 | `js/ui.js` |

### 그림(에셋)을 더 넣고 싶다면
지금은 이모지와 CSS만으로 화면을 그리고 있습니다. 직접 만든 그림을 넣으면 훨씬 풍성해집니다.
**[ASSETS.md](ASSETS.md)** 에 이미지 생성 AI용 프롬프트와 규격, 연결 방법을 정리해 뒀습니다.
그림이 없어도 게임은 그대로 작동하니 필요한 것만 하나씩 넣으시면 됩니다.

### 화면 디자인에 대해
- 바탕은 짙은 바다색, 카드는 양피지색으로 두어 **게임 화면 같은 분위기**를 만들었습니다.
- 자원은 등급별로 색이 다릅니다: <span>원자재(초록) · 가공품(파랑) · 완제품(보라)</span>.
  학생이 "이건 비싼 거구나"를 색으로 바로 알아챌 수 있습니다.
- 나라마다 **고유한 색의 문장(크레스트)** 이 자동으로 배정됩니다.
  윈도우 PC에서는 국기 이모지가 `KR`, `FR` 같은 두 글자로 보이는데, 색이 있어서 구분에 문제가 없습니다.
- 돈이 늘어나면 숫자가 올라가는 연출이 나오고, 이벤트는 **뉴스 속보**처럼 표시됩니다.
- 글꼴은 Pretendard를 인터넷에서 불러옵니다. 인터넷이 막힌 환경에서는 시스템 기본 한글 글꼴로 자동 대체됩니다.
- 「동작 줄이기」를 켠 기기에서는 애니메이션이 자동으로 꺼집니다.

> 나라와 특산품, 시작 자금은 **코드를 고치지 않고도** 방 만들기 화면에서 바꿀 수 있습니다.

## 자주 묻는 질문

**Q. 학생이 실수로 창을 닫았어요.**
같은 기기에서 다시 접속하면 원래 나라로 돌아갑니다. 다른 기기라면 코드와 이름을 다시 입력하고 같은 나라를 고르면 됩니다.

**Q. 모둠 인원이 몇 명까지 되나요?**
제한 없습니다. 팀 투표는 인원수에 맞춰 과반이 자동 계산됩니다.

**Q. 거래가 "체결 실패"로 떴어요.**
합의한 뒤 턴이 진행되기 전에 다른 거래로 자원이나 돈을 써버린 경우입니다. 실제 무역에서도 일어나는 일이니 수업 소재로 활용해 보세요.

**Q. 게임이 너무 빨리 끝나요 / 너무 느려요.**
턴 수와 시작 자금으로 조절하세요. 협상 시간이 길어지면 턴 진행을 조금 일찍 눌러도 됩니다.

**Q. 요금이 나오나요?**
Firebase 무료 등급(Spark)으로 한 학급 규모는 충분합니다. 하루 수업 정도로는 무료 한도의 극히 일부만 사용합니다.
